Dokumen ini ditujukan bagi developer pihak ketiga untuk mengintegrasikan sistem dengan aplikasi Kasir Percetakan.

1. Informasi Dasar

  • Base URL https://[domain-aplikasi-kasir]/api/v1
  • Format Data JSON
  • Metode HTTP GET (Saat ini hanya GET)

2. Autentikasi

Setiap request WAJIB menyertakan API Key pada header HTTP. Silahkan hubungi Admin untuk generate API Key

Header Nilai Deskripsi
X-API-Key [API_KEY_ANDA] Kunci akses unik
Accept application/json Memastikan response format JSON

3. Struktur Response Dasar

Secara umum, response dari API mengikuti format standar berikut (misal: Customer Lookup dan List Product):

{
  "status": "success" | "not_found" | "error",
  "message": "Deskripsi singkat hasil request",
  "data": { ... } | [ ... ] | null,
  "meta": { ... } // Opsional, hanya muncul pada data berhalaman (pagination)
}
Pengecualian: Endpoint Health Check, Smart Product Search, dan AI Recommend Engine memiliki struktur respons yang sedikit berbeda (flat hierarchy dengan key seperti "success": true) yang dioptimasi agar langsung dikonsumsi Frontend Chat interface / Agent AI. Silakan rujuk detail formatnya pada masing-masing endpoint di bawah.

4. Rate Limiting

  • Limit: 60 request per menit per API Key.
  • Retry-After: Jika melampaui batas, server mengembalikan status 429 Too Many Requests.

GET /customers/lookup

Mencari data pelanggan berdasarkan nomor WhatsApp/Handphone.

C. List Categories

GET /products/categories

Mengambil daftar kategori beserta keyword sinonim yang harus digunakan AI untuk memetakan pertanyaan pelanggan ke kategori yang benar sebelum memanggil `/products`.

Alur yang Benar untuk AI:
  1. Panggil `/products/categories` sekali saat inisiasi untuk melihat pola pemetaan.
  2. Cari kata yang diucapkan klien (misal: "spanduk") di field keywords tiap kategori.
  3. Gunakan id kategori yang ditemukan sebagai filter pada `/products`.

Response (200 OK)

{
  "status": "success",
  "data": [
    {
      "id": 1,
      "name": "Outdoor",
      "description": "Produk cetak OUTDOOR (luar ruangan)...",
      "keywords": ["spanduk", "banner", "flexy", "outdoor", ...],
      "price_type": "m2"
    },
    {
      "id": 7,
      "name": "Akrilik",
      "description": "Produk Akrilik & Sablon...",
      "keywords": ["akrilik", "acrylic", "sablon", "cetak dtf"],
      "price_type": "cm2"
    }
  ]
}

D. List & Search Products

GET /products

Mengambil daftar produk aktif. Perhatikan field price_type untuk perhitungan harga.

Parameter URL (Opsional)

Parameter Tipe Deskripsi
search String Cari berdasarkan Judul, Barcode, Kategori
category Mixed Filter ID Kategori (angka) atau Nama (teks)
per_page / page Integer Pagination kontrol (Max 100 per_page)
Logika Perhitungan Harga:

Selalu baca `price_formula` pada respon. Secara umum:

  • unit: Total = Jumlah x Price
  • m2: Total = (P_cm / 100) x (L_cm / 100) x Jumlah x Price
  • cm2: Total = P_cm x L_cm x Jumlah x Price

Response (200 OK)

{
  "status": "success",
  "data": [
    {
      "id": 1,
      "category_name": "Outdoor",
      "name": "Spanduk Glossy 280gr",
      "price": 20000,
      "price_type": "m2",
      "price_formula": "(Panjang_cm / 100) * (Lebar_cm / 100) * Jumlah * Harga_Jual",
      "unit": "PCS",
      "is_custom_size": true,
      "status": "aktif"
    }
  ],
  "meta": { "current_page": 1, "total": 120 }
}

E. Smart Product Search

AI Focused
GET /products/search

Pencarian cerdas yang dioptimalkan untuk AI Chatbot. Mendukung typo tolerance, sinonim, dan implicit category detection.

Parameter

  • q (Wajib): Kata kunci pencarian (Misal: "baner ultah")
  • per_page: Jumlah hasil (Default 10)
  • "baner" → "banner" (Typo)
  • "mmt" → "spanduk" (Synonym)
  • "spanduk" → Prioritas "Outdoor"
{
  "success": true,
  "query": "baner ultah",
  "normalized_query": "banner ulang tahun",
  "implied_category": "Outdoor",
  "best_match": {
    "id": 1,
    "name": "Spanduk Glossy 280 Gsm",
    "similarity_score": 92
  },
  "all_results": [ ... ]
}

F. AI Recommendation Engine

Sangat Disarankan
GET /ai/recommend

Endpoint ini otomatis membaca pesan bahasa manusia (natural language), mendeteksi tipe produk, mengekstrak dimensi ukuran (misal: "3x1 meter"), dan menghitung harga asli secara presisi. Sangat disarankan dipakai oleh Agent AI.

Parameter Utama

message

Pesan mentah pengguna.
Contoh: "Kak mau cetak spanduk ukuran 3x1 meter. Berapa harganya ya?"

Feedback AI: Endpoint ini menyediakan ai_instruction yang berisi prompt otomatis yang bisa langsung diteruskan ke LLM Anda agar jawabannya 100% akurat. 
{

  "customer_message": "kak mau cetak spanduk ukuran 3x1 meter bisa ga?",

      "width": 3,
      "height": 1,
      "unit": "meter",
      "area_m2": 3,
      "display": "3 × 1 meter"
  },
  "price_type": "m2",
  "price_examples": [
      {
          "product_name": "Spanduk Glossy 280 Gsm",
          "calculation": "3 m² × Rp 20.000 = Rp 60.000",
          "total_price": 60000
      }
  ],
  "ai_instruction": "INSTRUKSI UNTUK AI:\nPelanggan bertanya tentang produk kategori Outdoor..."
}

6. Kode Status HTTP & Error

Kode Status Deskripsi
200 OK Request berhasil diproses.
401 Unauthorized API Key salah, kadaluarsa, atau kosong.
404 Not Found Data tidak ditemukan.
422 Unprocessable Validasi parameter input gagal (contoh: format nomor salah).
429 Too Many Requests Melebihi limit rate API (60 req/menit).
500 Server Error Kesalahan internal server.

7. Troubleshooting & Kontak

Jika terjadi kendala integrasi, periksa hal berikut sebelum menghubungi admin:

  • Pastikan Base URL dan routing domain aplikasi sudah benar (tidak typo).
  • Pastikan header X-API-Key dikirim tanpa tambahan spasi.
  • Format nomor HP pada endpoint lookup idealnya angka (min 9 digit).
  • Cek konfigurasi HTTP Client Anda agar selalu menyematkan header Accept: application/json.
